## 1. 执行摘要

本文档定义B2B客户端健康数据仪表盘的产品方案，为健康管理机构、康复医院、健身器材厂商提供统一的数据可视化与运营管理平台。

**MVP目标**: 2周内实现可演示的前端原型，覆盖核心可视化场景。

## 3. 数据可视化需求清单

### 3.1 实时数据卡片 (P0)

| 卡片名称 | 数据指标 | 刷新频率 | 样式 |
|----------|----------|----------|------|
| 今日活跃用户 | 当前在线用户数 | 实时 | 数字+趋势箭头 |
| 设备在线率 | 在线设备数/总设备数 | 30秒 | 百分比+环形图 |
| 今日运动人次 | 运动干预执行次数 | 5分钟 | 数字+日环比 |
| 睡眠达标率 | 睡眠质量≥80分用户占比 | 5分钟 | 百分比+颜色指示 |

### 3.2 趋势图表 (P0)

| 图表类型 | 数据维度 | 时间范围 | 交互 |
|----------|----------|----------|------|
| 折线图 | 活跃用户数 | 7天/30天/自定义 | 悬停显示详情 |
| 面积图 | 健康指标趋势(心率/睡眠) | 7天/30天 | 支持时间范围选择 |
| 柱状图 | 各时段运动量分布 | 今日/本周 | 支持钻取到用户 |
| 热力图 | 用户活跃时段分布 | 本周 | - |

### 3.3 告警面板 (P1)

| 告警类型 | 触发条件 | 展示位置 | 颜色 |
|----------|----------|----------|------|
| 设备离线 | 设备心跳超时>60秒 | 设备监控页顶部 | 红色 |
| 异常生理指标 | 心率>180或<40 | 用户详情页 | 橙色 |
| 方案依从性低 | 连续3天未执行 | 干预追踪页 | 黄色 |
| 数据质量异常 | 数据完整率<50% | 首页概览 | 蓝色 |

## 4. 交互流程设计

### 4.1 筛选流程

```
用户选择筛选条件 → 实时更新数据 → 支持保存筛选预设
     ↓
筛选维度: 时间范围 / 机构 / 设备类型 / 用户群体 / 干预类型
```

### 4.2 钻取流程

```
首页KPI卡片 → 点击进入详情页 → 选择用户/设备 → 查看详细数据
     ↓
示例: 今日运动人次(500) → 点击 → 运动记录列表 → 选择某用户 → 查看该用户运动详情
```

### 4.3 导出流程

```
选择数据范围 → 选择导出格式(CSV/Excel/PDF) → 生成报表 → 下载
     ↓
支持导出: 用户健康数据 / 设备运行报表 / 干预效果汇总
```

## 6. 用户故事细化 (US-V系列)

### US-V-001: 首页概览数据卡片

**优先级**: P0

**用户故事**:
> 作为机构管理员，我需要在首页看到关键业务指标的一览数据，以便快速了解机构运营状态。

**验收标准**:
- Given 用户登录系统并进入首页
- When 页面加载时
- Then 显示4个核心KPI卡片：今日活跃用户、设备在线率、今日运动人次、睡眠达标率
- And 每个卡片显示实时数值和趋势指示（上升/下降/持平）

- Given 设备在线率低于50%
- When 页面渲染时
- Then 设备在线率卡片显示为橙色警告样式

- Given 用户点击某个KPI卡片
- When 卡片被点击时
- Then 跳转到对应详情页面

### US-V-002: 设备在线状态监控

**优先级**: P0

**用户故事**:
> 作为机构管理员，我需要实时监控所有设备的在线状态，以便及时发现和处理设备故障。

**验收标准**:
- Given 管理员进入设备监控页面
- When 页面加载时
- Then 显示设备列表，列：设备名称、类型、状态、最后在线时间、关联用户数
- And 状态筛选器：全部/在线/离线/故障

- Given 设备心跳超时超过60秒
- When 系统检测到时
- Then 设备状态自动更新为"离线"
- And 页面顶部显示离线告警横幅

- Given 管理员点击设备行
- When 设备行被点击时
- Then 展开显示设备详情：基本信息、连接历史、近7天数据上报统计

### US-V-003: 用户健康档案查看

**优先级**: P0

**用户故事**:
> 作为健康师，我需要查看用户的健康档案和相关数据，以便为用户提供个性化干预服务。

**验收标准**:
- Given 健康师进入用户管理页面
- When 页面加载时
- Then 显示用户列表，列：姓名、注册时间、关联设备数、健康画像状态
- And 支持搜索用户（按姓名、手机号）
- And 支持按健康画像状态筛选（未创建/采集中/已完成）

- Given 健康师点击用户行
- When 用户行被点击时
- Then 展开显示用户健康档案：基本信息、健康画像5维度评分、关联设备、最近干预方案

- Given 用户有异常生理指标
- When 健康师查看用户详情时
- Then 显示异常指标告警标识，点击可查看异常详情

### US-V-004: 干预方案执行追踪

**优先级**: P1

**用户故事**:
> 作为机构管理员，我需要追踪用户对干预方案的执行情况，以便评估干预效果。

**验收标准**:
- Given 管理员进入干预追踪页面
- When 页面加载时
- Then 显示干预方案列表，列：方案类型、目标用户、执行状态、依从性、完成率

- Given 用户连续3天未执行方案
- When 页面渲染时
- Then 该用户在列表中显示低依从性标识（橙色）
- And 页面顶部显示依从性告警

- Given 管理员选择查看某个方案详情
- When 点击方案行时
- Then 展开显示方案执行明细：每日执行记录、生理指标变化图表、效果评估

### US-V-005: 数据导出

**优先级**: P1

**用户故事**:
> 作为机构管理员，我需要导出健康数据报表，以便进行离线分析和汇报。

**验收标准**:
- Given 管理员进入报表页面
- When 选择数据范围和导出格式时
- Then 支持选择：用户健康数据/设备运行报表/干预效果汇总
- And 支持格式：CSV、Excel、PDF

- Given 管理员点击导出按钮
- When 导出请求发送时
- Then 显示导出进度
- And 完成后自动下载文件

- Given 数据量较大（>10000条）
- When 导出请求发送时
- Then 后台处理并发送下载链接到邮箱

### US-V-006: 实时数据趋势查看

**优先级**: P0

**用户故事**:
> 作为机构管理员，我需要查看健康数据的趋势变化，以便分析业务发展情况。

**验收标准**:
- Given 管理员在首页查看趋势图
- When 选择时间范围（7天/30天/自定义）时
- Then 图表自动更新显示对应时间范围数据
- And 悬停显示具体数值

- Given 管理员查看用户活跃时段分布
- When 选择热力图时
- Then 显示一周各时段的活跃用户数热力图
- And 颜色越深表示活跃用户越多

### US-V-007: 用户权限管理

**优先级**: P1

**用户故事**:
> 作为机构管理员，我需要管理系统用户的权限，以便控制数据访问范围。

**验收标准**:
- Given 管理员进入系统设置页面
- When 查看用户列表时
- Then 显示系统用户列表，列：姓名、角色、最后登录时间、状态

- Given 管理员创建新用户
- When 填写用户信息并选择角色时
- Then 系统创建用户账号
- And 发送初始密码到用户邮箱

- Given 管理员修改用户角色
- When 选择新角色并确认时
- Then 用户权限立即更新

## 7. 技术选型建议

### 7.1 前端框架

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| React + Ant Design | 组件丰富，适合后台系统 | 学习曲线 | ⭐⭐⭐⭐⭐ |
| Vue3 + Element Plus | 上手快，文档友好 | 生态略弱于React | ⭐⭐⭐⭐ |
| React + Material UI | 国际化好 | 体积较大 | ⭐⭐⭐ |

**推荐**: React + Ant Design Pro (开箱即用的中后台方案)

### 7.2 图表库选型

| 库 | 特点 | 适用场景 |
|----|------|----------|
| ECharts | 图表丰富，文档好 | 复杂图表、地图 |
| AntV/G2 | 高度可定制 | 业务图表 |
| Recharts | React原生 | 简单图表 |

**推荐**: ECharts (满足所有可视化需求，社区活跃)

### 7.3 实时数据更新方案

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| WebSocket | 双向通信、低延迟 | 需要维护长连接 | 设备状态实时监控 |
| SSE | 简单、单向 | 不支持跨域 | 数据趋势推送 |
| Polling | 实现简单 | 有延迟 | 低频数据更新 |

**推荐**: 
- 设备监控页面: WebSocket
- KPI卡片: SSE或30秒轮询
- 其他页面: 按需加载

## 9. API对接需求

### 9.1 现有API (device-integration/api)

根据PRD_MVP_v1.0.md第5.2节，需对接以下API：

```bash
# 设备相关
GET  /api/devices              # 设备列表
GET  /api/devices/{id}         # 设备详情
GET  /api/devices/{id}/data    # 设备数据

# 用户相关
GET  /api/users                # 用户列表
GET  /api/users/{id}/profile   # 用户画像
GET  /api/users/{id}/metrics   # 健康指标

# 干预相关
GET  /api/users/{id}/prescriptions    # 干预方案列表
GET  /api/users/{id}/reports/weekly   # 周报
GET  /api/users/{id}/reports/monthly  # 月报
```
