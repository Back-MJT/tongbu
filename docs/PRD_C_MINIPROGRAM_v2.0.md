## 0. 执行摘要

HealthHub C端小程序 v2.0 是适配 Board v1.1 战略转向的微信小程序产品需求文档，定义为**健身器械行业数字神经系统的用户触点**。

**核心变化（对比v1.0）**:

| 维度 | v1.0（旧） | v2.0（新版） |
|------|-----------|-------------|
| 目标用户 | 健康管理机构用户 | 健身器械厂家设备用户 |
| 数据来源 | 穿戴设备 + 手动记录 | IMU+BLE模组自动采集 |
| 核心场景 | 运动 + 睡眠 + 营养 + 压力 + 心肺 | 运动专项（力量/耐力/柔韧/核心/心肺） |
| 用户入口 | 机构邀请码 | 扫描设备二维码 |
| 健康画像 | 5维度雷达图 | 运动能力专项雷达 |
| 设备绑定 | 穿戴设备配对 | BLE扫码连接器械模组 |

**MVP目标**: 6周内小程序上线，配合M3厂家设备出货，触达首批种子用户。

**核心价值**: 组数记录自动化 + AI个性化训练方案 + 进度可视化依从性追踪。

## 1. 产品定位

### 1.1 在四层产品体系中的位置

```
第零层: IMU+BLE硬件模组（传感器 + 数据广播）
        ↓ BLE
第一层: 设备数字化层（Device SDK + 厂家后台）
        ↓ MQTT
第二层: AI训练引擎层（Claude API → 个性化训练方案）
        ↓
第三层: 数据价值层（行业对标报告）
        ↓
C端微信小程序 ← 这是本文档定义的产品
```

小程序是第二层（AI训练引擎）的用户触点，接收IMU模组数据，调用Claude API，呈现个性化训练方案。

### 1.2 目标用户画像

**主用户群**: 购买宁津县健身器械厂家设备的个人用户（家用/小型商用）

| 属性 | 描述 |
|------|------|
| 年龄 | 25-50岁 |
| 场景 | 家用综合训练架、史密斯机、绳索机等抗阻设备 |
| 痛点 | 不知道练什么、组数靠记忆、无法追踪长期进步 |
| 期望 | 一个私教级的智能训练助手，不用改变习惯 |

### 1.3 核心价值主张

> **"你的器械变聪明了"** — IMU模组自动记录组数，AI生成个性化训练方案，进度一目了然。

## 3. 核心功能定义

### 3.1 设备绑定（扫码 + BLE）

**场景**: 用户购买带IMU模组的健身器械后，微信扫码完成绑定

**流程**:
```
用户打开小程序 → [首页/我的]点击"扫码绑定"
    → 微信扫码（扫描设备二维码）
    → 小程序通过BLE搜索附近设备
    → 用户选择对应设备点击连接
    → 连接成功，显示设备信息
    → 绑定完成，跳转首页
```

**BLE绑定数据流**:
```
IMU模组（设备侧）
    ↓ BLE广播
微信小程序（用户侧）BLE模块
    ↓ 解析数据
本地缓存（小程序 storage）
    ↓ 训练结束/有网络时上传
云端 MQTT Broker
    ↓
Claude API（训练方案生成）
```

### 3.2 组数自动记录

IMU模组固件级算法自动检测并上报组数，小程序接收展示：

| 场景 | 显示内容 |
|------|---------|
| 正在进行 | 实时组数更新（每完成一组刷新） |
| 今日目标 | "[已完成的组数] / [目标组数] [进行中/达标]" |
| 组间休息 | 显示距下一组开始倒计时 |
| 训练结束 | 显示今日总结：总组数、总时长、消耗估算 |

### 3.3 AI个性化训练方案

**触发时机**: 每次打开首页 + 每次完成一组训练后刷新建议

**数据输入到Claude API**:
```
用户：{昵称}，{年龄}岁，{性别}
设备：{器械类型}，型号：{型号}
目标：{运动目标：增肌/塑形/维持}
过去30天：训练{N}次，平均{M}分钟/次
最近一次：{日期}，完成{X}组/目标{Y}组
当前阶段：{初学/成长/停滞/进阶}

请用中文给出：
1. 今日训练建议（针对当前设备，3条具体动作，含组数/次数推荐）
2. 一句话鼓励语（像私教朋友）
```

**Claude API调用规范**（复用intervention-engine callClaude()函数）:
- temperature=0，保证一致性
- 缓存：同一用户数据未变化时60分钟内复用缓存
- 日志：记录prompt、response、token消耗、延迟

### 3.4 每日训练任务

**任务生成**: Claude API根据用户档案和历史数据生成

**任务数据结构**:
```json
{
  "task_id": "uuid",
  "date": "2026-04-14",
  "device_id": "LK-2024-0012",
  "exercise_type": "胸肌训练",
  "prescription": [
    {"name": "平板卧推", "sets": 4, "reps": 12, "rest_sec": 90},
    {"name": "上斜哑铃卧推", "sets": 3, "reps": 10, "rest_sec": 90},
    {"name": "绳索飞鸟", "sets": 3, "reps": 15, "rest_sec": 60}
  ],
  "status": "in_progress",
  "completed_sets": 4,
  "total_sets": 10
}
```

### 3.5 进度追踪与成就系统

**依从率计算**: 已完成组数 / 处方总组数 × 100%

**成就徽章体系**:

| 徽章 | 触发条件 | 效果 |
|------|---------|------|
| 初学者 | 完成首次训练 | 激励继续 |
| 3日连续 | 连续3天有训练 | 强化习惯 |
| 7日连续 | 连续7天有训练 | 里程碑 |
| 10日连续 | 连续10天有训练 | 黄金习惯 |
| 力量新秀 | 单日完成10+组 | 突破激励 |
| 30日坚持 | 30天累计训练 | 长期认可 |

### 3.6 运动能力雷达图

**替代v1.0的五维健康雷达（运动/睡眠/营养/压力/心肺）**

v2.0专注运动专项五维度:

| 维度 | 评估方式 | 数据来源 |
|------|---------|---------|
| 力量 | 负重动作组数 × 估算重量等级 | IMU振动强度 |
| 耐力 | 每次训练持续时长 × 频率 | IMU使用时长 |
| 柔韧 | 用户自评问卷（首次建档） | 用户输入 |
| 核心 | 核心稳定动作完成质量 | IMU振动模式 |
| 心肺 | 有氧训练频率（暂时无氧设备不计入） | IMU使用检测 |

## 4. 用户故事 (W3-US-P系列)

> 编号说明: W3 = Wave 3 (v1.1执行阶段), US-P = User Story Product (小程序端)

### US-P-001: 扫码绑定设备

**优先级**: P0

**用户故事**:
> 作为新用户，我需要通过扫描设备二维码绑定我的健身器械，以便开始自动记录训练数据。

**验收标准**:
- Given 用户首次打开小程序且未绑定设备
- When 页面加载时
- Then 显示"扫码绑定设备"引导入口

- Given 用户点击"扫码绑定"
- When 微信扫码后
- Then 小程序搜索附近BLE设备
- And 显示设备列表（名称 + 信号强度）

- Given 用户选择对应设备并点击连接
- When BLE连接成功时
- Then 显示绑定成功提示
- And 设备信息（名称、型号、安装日期）展示在"我的设备"中

- Given 用户绑定成功后
- When 跳转首页时
- Then 今日训练卡片显示该设备信息

### US-P-002: 首页训练看板

**优先级**: P0

**用户故事**:
> 作为用户，我需要一眼看到今日训练进度和AI建议，以便快速决定今天练什么。

**验收标准**:
- Given 用户打开小程序并进入首页
- When 页面加载时
- Then 显示今日训练卡片：设备名称、已完成组数/目标组数、状态标识

- Given 用户当日训练进行中
- Then 卡片显示"[进行中 ●]"状态
- And 实时更新已记录组数

- Given 用户当日训练已完成（达标）
- Then 卡片显示"[达标 ✓]"状态
- And 显示绿色标识

- Given 用户当日未开始训练
- Then 卡片显示"[未开始]"状态
- And AI建议区域显示今日推荐训练

- Given 用户当日训练未达标
- Then 卡片显示"[未达标 △]"状态
- And AI建议区域提示补练

### US-P-003: 组数自动记录

**优先级**: P0

**用户故事**:
> 作为用户，我不需要手动记录组数，IMU模组自动检测并上报组数据，让训练更专注。

**验收标准**:
- Given 用户在训练中，设备已连接
- When IMU检测到一组动作完成时
- Then 小程序实时刷新已记录组数（+1）
- And 震动提示（用户授权时）

- Given 用户在组间休息
- Then 显示距下一组开始倒计时（IMU检测动作重新开始计时）

- Given 用户完成今日训练
- When IMU检测30秒无动作时
- Then 触发训练结束判断
- And 显示今日训练总结弹窗（总组数、总时长、消耗估算）

### US-P-004: 每日训练任务

**优先级**: P0

**用户故事**:
> 作为用户，我需要清晰的每日训练处方，知道每个动作做几组几次，并能在小程序内追踪完成状态。

**验收标准**:
- Given 用户进入每日任务页
- When 页面加载时
- Then 显示当日训练处方：动作名称、组数、次数、休息时长

- Given 用户完成一组动作（IMU自动检测）
- When 组数上报时
- Then 对应动作的"已完成组数"刷新

- Given 用户点击某个动作
- Then 显示动作详情：名称、组数×次数、休息间隔、动作要点

- Given 用户完成处方中所有动作组数
- Then 该动作显示"完成 ✓"状态
- And 显示下一动作推荐

### US-P-005: AI训练建议

**优先级**: P0

**用户故事**:
> 作为用户，我需要AI根据我的训练历史和当前阶段，生成个性化的本周训练建议，像有个私教在指导。

**验收标准**:
- Given 用户进入首页
- When 页面加载时
- Then AI建议区域显示Claude API生成的本周训练建议（3条）

- Given 用户上周训练强度下降
- When AI分析时
- Then 建议中包含"增加训练频率"类提醒

- Given 用户处于初学期（<4周）
- When AI生成建议时
- Then 建议偏重基础动作和正向激励，避免高难度动作

- Given 用户处于进阶期（3个月+）
- When AI生成建议时
- Then 建议包含周期化训练和竞技目标

### US-P-006: 进度追踪

**优先级**: P0

**用户故事**:
> 作为用户，我需要看到自己的长期训练依从性和进步趋势，以获得持续训练的动力。

**验收标准**:
- Given 用户进入进度追踪页
- When 页面加载时
- Then 显示本月依从率（已完成组数 / 处方总组数）
- And 显示连续坚持训练天数

- Given 用户点击"运动能力雷达图"
- Then 展开五维雷达图（力量/耐力/柔韧/核心/心肺）

- Given 用户完成30天训练
- When 进入进度页时
- Then 显示"30日坚持"成就徽章

### US-P-007: 成就徽章

**优先级**: P0

**用户故事**:
> 作为用户，我需要通过成就徽章获得正反馈，让训练变得像游戏一样有目标和奖励。

**验收标准**:
- Given 用户达成徽章解锁条件
- When 进入进度页时
- Then 显示徽章获得动画
- And 记录徽章获得时间

- Given 用户点击已有徽章
- Then 显示徽章详情：获得时间、对应成就条件

- Given 用户连续3天有训练
- When 第3天训练完成时
- Then 解锁"3日连续"徽章

### US-P-008: 用户档案与设备管理

**优先级**: P0

**用户故事**:
> 作为用户，我需要管理我的个人信息和已绑定设备，确保数据准确。

**验收标准**:
- Given 用户进入"我的档案"页
- When 页面加载时
- Then 显示：头像、昵称、当前训练阶段

- Given 用户点击"已绑定设备"
- Then 展示设备列表：设备名称、型号、绑定时间、在线状态

- Given 用户点击某设备
- Then 显示设备详情：型号、固件版本、信号状态

- Given 用户点击"扫码绑定新设备"
- Then 跳转扫码绑定流程（同US-P-001）

### US-P-009: 设备解绑

**优先级**: P1

**用户故事**:
> 作为用户，我需要能将设备解除绑定，以便换到其他设备或重新绑定。

**验收标准**:
- Given 用户在设备详情页点击"解除绑定"
- When 点击时
- Then 显示确认弹窗："确定解除绑定吗？"

- Given 用户确认解除绑定
- Then 设备从列表移除
- And 本地缓存数据保留（历史训练记录）

### US-P-010: 通知提醒

**优先级**: P1

**用户故事**:
> 作为用户，我需要收到每日的训练提醒，帮助我保持训练习惯。

**验收标准**:
- Given 用户已开启通知权限
- When 每日早9点时
- Then 发送订阅消息：今日训练提醒（设备名称 + 目标组数）

- Given 用户在设置页关闭通知
- When 系统尝试推送时
- Then 不发送通知

## 5. 数据模型

### 5.1 核心实体

```typescript
// 用户
User {
  id: string (openid)
  nickname: string
```

### 6.3 BLE通信协议

**IMU模组广播数据格式** (复用 DEVICE_SCHEMA_UNIFIED):

```json
{
  "device_id": "LK-2024-0012",
  "type": "imu_data",
  "timestamp": 1713081600000,
  "data": {
    "accel": [x, y, z],
    "activity_type": "rep_detected",
    "rep_count": 4,
    "set_id": "set_001",
    "intensity": "medium"
  }
}
```

**小程序BLE接收处理**:
1. 监听 `wx.onBLECharacteristicValueChange`
2. 解析JSON数据
3. 更新本地缓存（`cached_prescription`）
4. 本地展示刷新
5. 训练结束时批量上传

### 6.4 Claude API调用流程

```
前端请求 → /api/llm/training_suggestion
    → 后端 callClaude() 函数
    → 检查Redis缓存（key: user_id + date）
    → 缓存命中 → 直接返回
    → 缓存未命中 → 调用Claude API
    → 缓存结果（TTL: 60分钟）
    → 返回结果
```

### 6.5 性能约束

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 3秒 (4G) |
| BLE连接 | < 5秒 |
| 页面切换 | < 300ms |
| 组数刷新延迟 | < 1秒（BLE数据到UI） |
| 小程序包体积 | < 2MB |
