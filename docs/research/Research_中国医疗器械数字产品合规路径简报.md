## 执行摘要

Phase 3启动前提之一是"合规扫清障碍"。本简报回答：昕动智能的**数字底座+干预引擎**产品组合，在中国监管框架下处于什么位置？需要什么资质？按什么路径合规？

**核心结论**：

| 产品模块 | 监管分类 | 注册路径 | 难度 |
|---------|---------|---------|-----|
| 健康数据记录软件（不含诊断建议） | Ⅰ类器械 | 备案制，90日 | 低 |
| 健康管理APP/小程序（仅健康教育/提醒） | 信息服务，不需器械注册 | ICP备案+网络安全等保 | 低 |
| **运动处方生成软件（软件驱动的干预方案）** | **Ⅱ类器械（待确认）** | **注册检测+临床评价+注册审评，约18-30个月** | **高** |
| **睡眠/压力干预算法软件** | **Ⅱ类或Ⅲ类器械（取决于适应症声明）** | **同上** | **高** |
| 可穿戴健康监测设备（心率/血氧/加速度） | Ⅱ类器械（如声称医疗用途） | 注册检测+临床评价 | 中-高 |

> **战略含义**：以"健康管理工具"而非"医疗器械"定位产品，可规避高成本注册路径，优先跑通B2B2C商业模式。但若目标进入医保/医院市场，则必须获取器械注册。本简报建议：**双轨策略——消费级产品先行，医疗级产品按需注册。**

## 一、监管框架概览

### 1.1 主管机构

- **NMPA（国家药品监督管理局）**：医疗器械注册与监管
- **国家卫健委**：互联网医疗、远程医疗政策
- **工信部**：软件和信息服务业管理、APP备案
- **公安部**：信息安全等级保护
- **网信办**：个人信息保护、数据安全

### 1.2 器械分类规则

依据《医疗器械分类规则》（总局令第15号）和《医疗器械分类目录》（2017版）：

| 类别 | 风险等级 | 注册方式 | 典型产品 |
|------|---------|---------|---------|
| **Ⅰ类** | 低风险 | **备案制**（市级药监局） | 医用绷带、手术手套 |
| **Ⅱ类** | 中等风险 | **注册制**（省级药监局检测+审评） | 血压计、心电图机、医用口罩 |
| **Ⅲ类** | 高风险 | **注册制**（国家药监局审评） | 心脏起搏器、人工关节、ICU设备 |

**数字产品关键判断标准**：
> 产品的预期用途是否具有**诊断、治疗、缓解、预防疾病**的功能？若是，则按医疗器械管理。

## 二、数字健康产品分类判断

### 2.1 软件分类决策树

```
软件功能判断：
├── 是否用于辅助诊断、治疗决策？
│   ├── 是 → 可能属Ⅱ类或Ⅲ类器械
│   └── 否
├── 是否提供个性化医疗建议/处方？
│   ├── 是 → 可能属Ⅱ类器械（SaMD）
│   └── 否
└── 仅提供健康教育、数据记录、提醒？
    → 不属于医疗器械（按信息服务管理）
```

### 2.2 已有监管先例（2022-2024）

| 产品 | 公司 | 监管类别 | 路径 |
|------|------|---------|-----|
| 平安好医生慢病管理APP | 平安健康 | Ⅱ类器械（慢病管理） | NMPA注册 |
| 杏树林医药云 | 杏树林 | Ⅱ类器械（患者管理） | NMPA注册 |
| 腾讯觅影（AI辅助诊断） | 腾讯 | Ⅲ类器械（AI诊断） | 国家药监局创新通道 |
| 腕上心电记录设备 | 华为/小米 | Ⅱ类器械（心电记录） | 注册已获批 |
| 呼吸睡眠监测APP（消费级） | 多家 | 不需要器械注册 | 仅有APP备案 |

### 2.3 运动处方/睡眠干预软件——最相关参考

**国家药监局2022年发布的《医疗器械分类目录》调整**：
- "02-07-03 康复训练软件"：用于运动康复训练，**Ⅱ类器械**
- "21-06 决策支持软件"：具有诊断/治疗建议功能 → **Ⅱ类或Ⅲ类**

**关键风险点**：
如果干预引擎生成的是"运动处方"而非"运动建议"，监管机构可能认定其属于医疗决策软件，需要Ⅱ类或更高器械注册。

**参考文件**：
- NMPA《医疗器械软件注册审查指导原则》（2022年修订版）
- 国家药监局《人工智能医疗器械注册审查指导原则》（2022年）

## 三、合规路径分析

### 3.1 路径一：消费级健康管理工具（推荐近期采用）

**适用场景**：APP/小程序形态的健康管理服务，不声称诊断/治疗功能

**合规要求**：

| 事项 | 要求 | 负责部门 |
|------|------|---------|
| ICP备案 | 工信部ICP/IP地址备案系统 | 公司自行 |
| 网络安全等保 | **二级等保**（建议三级） | 第三方测评机构 |
| 个人信息保护 | 《个人信息保护法》合规：告知同意、隐私政策、账号注销权 | 公司自行 |
| 内容合规 | 不得发布医疗效果承诺、虚假夸大内容 | 网信办/市监局 |
| 微信小程序合规 | 遵守微信小程序审核规范（医疗健康类需特殊资质） | 腾讯审核 |

**时间成本**：3-6个月，费用约5-15万元（含等保测评）
**无需NMPA器械注册**

### 3.2 路径二：Ⅱ类医疗器械注册（中期目标）

**适用场景**：产品进入医院/康复机构/医保报销场景，或明确声称干预效果

**注册流程**：

```
Step 1: 产品技术要求编制 → 2-4周
Step 2: 注册检验（NMPA认可检验机构） → 3-6个月
Step 3: 临床评价（免临床路径或临床试验） → 6-18个月
Step 4: 注册申报资料整理 → 2-4周
Step 5: 药监局审评审批 → 12-18个月
Step 6: 取得注册证 → 生产许可申请 → 3-6个月
─────────────────────────────────────────
合计：约 18-36 个月
```

**关键文件**：
- 产品技术要求（含性能指标、检测方法）
- 说明书和标签样稿
- 研究资料（软件生命周期文档、网络安全描述）
- 临床评价资料（免临床路径：同品种比对或临床文献）
- 质量管理体系文件（GMP）

**费用估算**：
- 注册检验：15-40万元
- 临床评价（如需）：30-100万元
- 注册代理/咨询：20-60万元
- 体系认证（GMP）：15-30万元
- **总计约80-230万元**

### 3.3 路径三：软件作为医疗器械（SaMD）快速通道

**创新器械特别审查**：
- NMPA设有**创新医疗器械特别审查通道**
- 条件：核心算法国际领先、有发明专利、有显著临床价值
- 优势：优先检测、优先审评、附条件审批
- 昕动干预引擎若在某个细分（如运动处方AI）有独特创新，可尝试此通道

**参考案例**：
- 腾讯觅影（AI眼底病变诊断）通过创新通道，审评周期压缩至约18个月

## 四、中国健康数据标准与互操作性

### 4.1 相关国家标准

| 标准号 | 名称 | 适用场景 | 与昕动关系 |
|--------|------|---------|-----------|
| GB/T 33136-2016 | 信息技术 学习、教育和培训 数据模型 | 教育培训数据 | 参考（非直接） |
| WS/T 601-2022 | 居民健康档案数据集 | 健康档案数据交换 | **核心参考** |
| WS/T 500-2016 | 电子病历数据集 | 病历数据标准化 | 参考 |
| GB/T 35273-2020 | 信息安全技术 个人信息安全规范 | 隐私保护 | **直接适用** |
| HL7 FHIR R4 | 国际医疗数据交换标准 | 数据互操作 | **架构参考** |

**WS/T 601-2022《居民健康档案数据集》**：
- 规定了健康档案的数据集结构，包括基本信息、疾病史、用药史、健康体检等
- 与昕动数字底座的数据模型直接对应
- **建议：数字底座数据模型与WS/T 601对齐，便于与医院/体检机构系统对接**

### 4.2 互操作路线建议

Phase 3的**健康装备数据接入规范白皮书**应与WS/T 601对齐：

1. **设备数据 → FHIR兼容格式**：设备原始数据 → 转换为FHIR Observation资源
2. **干预处方 → FHIR CarePlan资源**：运动处方、睡眠建议映射为标准FHIR CarePlan
3. **健康评估 → FHIR RiskAssessment资源**：健康评分可映射为FHIR风险评估

**参考**：
- 国家卫健委正在推进FHIR在中国的本地化落地
- 2023年国家卫健委规划司《全民健康信息平台建设指南》明确提及FHIR

## 五、信息安全三级等保

Phase 3要求"信息安全三级等保"（P1）。以下是对我们的具体影响：

### 5.1 等保2.0概述

等保2.0（《网络安全等级保护基本要求》GB/T 22239-2019）将等级分为五级：

| 等级 | 适用系统 | 监管层级 | 典型要求 |
|------|---------|---------|---------|
| 一级 | 私有网络 | 自主保护 | 基础防护 |
| 二级 | 一般企业系统 | 县级公安备案 | 增强防护，**建议健康APP达到** |
| **三级** | 重要系统/数据 | 市级公安+省厅监管 | **强烈建议**，含数据加密、访问审计、渗透测试 |
| 四级 | 关键基础设施 | 国家级监管 | 实时监控、紧急响应 |
| 五级 | 国防军事 | 国家级监管 | 最高防护 |

### 5.2 健康数据的特殊要求

由于处理**个人健康数据**（《个人信息保护法》定义为敏感个人信息），监管通常要求达到三级等保：

**三级等保核心控制点**：
- 身份鉴别：双因素认证
- 访问控制：基于角色的细粒度授权
- 安全审计：日志留存≥6个月
- 数据保密性：传输和存储加密（推荐国密SM2/SM4）
- 数据完整性：防止未授权篡改
- 数据备份：本地+异地容灾

### 5.3 费用与时间

| 项目 | 三级等保费用 | 时间 |
|------|------------|------|
| 测评机构 | 8-15万元/次 | - |
| 整改建设 | 20-80万元 | 2-4个月 |
| 复测 | 5-10万元 | 1-2个月 |
| **合计** | **约33-105万元** | **4-8个月** |

## 六、合规策略建议

### 6.1 近期（12个月内）：消费级产品跑通模式

**策略**：将APP/小程序定位为"健康管理工具"，规避器械注册

**行动项**：
- [ ] 完成ICP备案和二级等保（建议直接做三级）
- [ ] 完善《个人信息保护法》合规：隐私政策、用户协议、账号注销机制
- [ ] 微信小程序健康类目申请（需提供营业执照+相关资质承诺书）
- [ ] 产品文案规避医疗效果声明，用"改善"而非"治疗"
- [ ] 数据存储在中国大陆境内（网络安全法要求）

### 6.2 中期（12-24个月）：按需启动器械注册

**触发条件**：
- B端机构客户要求产品进入医院系统
- 医保/商保结算需要器械证
- 竞品已获得注册证形成壁垒

**行动项**：
- [ ] 确定核心产品模块的器械分类（与NMPA沟通确认）
- [ ] 启动注册检验（如决定走器械路径）
- [ ] 准备临床评价资料
- [ ] 建立质量管理体系（GMP）

### 6.3 长期（24个月+）：打造合规竞争力

**差异化机会**：
- 行业数据标准话语权：主导或参与"运动康复数据标准"团体标准制定
- FHIR本地化：提前布局FHIR兼容架构，形成与医院系统对接的技术壁垒
- 等保三级认证：作为B端客户选择供应商的重要加分项

## 九、结论：这对我们的产品意味着什么

**对于数字底座（设备接入+数据管道）**：
- 定位为"数据服务"而非"诊断工具" → 无需器械注册
- 合规重点：等保 + 个人信息保护 + 数据本地化

**对于干预引擎（运动处方/睡眠/营养/压力干预）**：
- 若定位为"健康建议/个性化方案" → 消费级产品路径，合规成本低
- 若定位为"临床干预方案" → 需要Ⅱ类器械注册（高成本、长周期）
- **建议：产品双版本策略——消费版（健康建议）和机构版（按需注册）**

**对于BD和销售**：
- 向客户说明："我们按消费级健康管理工具运营，正在布局医疗级产品线"
- 竞品如声称"数字疗法"但无注册证，则存在合规风险——可作为竞争差异化点
