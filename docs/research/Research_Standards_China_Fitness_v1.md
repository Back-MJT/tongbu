# 中国健身器械数字化标准体系 — 情报简报

> 昕动智能 · Health Tech Researcher · Standing Task 产出
> 版本: v1.0 · 2026-04-15
> 用途: BD团队参展宁津体博会应对"合规/标准"质疑；证明HealthHub是国内健身器械数据标准的合规底座

---

## 执行摘要

**核心发现**:
1. 中国健身器械数据格式标准仍在空白期——这是窗口机遇，谁先做标杆谁参与定义标准
2. GB/T 35273（个人信息安全）和等保2.0是健身数据合规底线，必须满足
3. 出口欧洲CE/美国FCC认证是BLE模组的准入条件
4. FHIR Device/Observation是HealthHub与国家全民健康信息平台对接的技术锚点
5. 协会/地方政府正在推动"智慧体育器材"地方标准——宁津厂商有机会参与制定

**对BD的核心话术**:
> "中国健身器材数字化标准现在是窗口期——国家标准层面数据格式规范还是空白。个人数据安全(GB/T 35273)和出口合规(CE/FCC)已经是硬门槛。我们从第一天就按FHIR国际标准设计，满足这些要求。更重要的是：谁先参与制定地方标准，谁就是未来行业标准的定义者。"

## 一、标准化行政管理架构

```
国家体育总局 → 全国体育标准化技术委员会（TC 456）
  └── 健身器材分技委（SC 1）—— 与数字化直接相关
  └── 运动场地设施分技委（SC 3）
  └── 运动服装/鞋类分技委（SC 2）
```

## 二、现行健身器材相关GB/T标准

| 标准号 | 名称 | 状态 | 数字化相关度 |
|--------|------|------|------------|
| GB 17498.1-2008 | 固定式健身车 安全要求 | 现行 | 无（仅机械安全） |
| GB 19272-2011 | 跑步机 安全要求 | 现行（修订中） | 无 |
| GB/T 41134-2021 | 体育设施 场地数据分类与编码 | 现行 | 可类比参考 |
| GB/T 39599-2020 | 体育用品 健身综合训练器 | 现行 | 机械性能，无数据要求 |
| GB/T 41135-2021 | 健身器材使用说明 | 现行 | 纸质说明要求，无数据 |
| GB/T 41134.1-2021 | 体育用品分类与编码 | 现行 | 仅术语 |

**关键判断**: 健身器械的**数据格式、传输协议、互操作性标准，在国家标准层面仍是空白**。这是窗口机遇。

## 三、健身数据合规底线（必须满足）

### 3.1 GB/T 35273-2020 个人信息安全规范

| 数据类型 | 分类 | 合规要求 |
|---------|------|---------|
| 用户身份（姓名、手机、年龄） | 个人信息 | 收集必须明示同意 |
| 运动数据（心率、姿势、使用时长） | 个人信息（健康数据级别）| 需单独授权 |
| 设备标识（MAC地址） | 个人信息 | 需加密存储 |

### 3.2 等保2.0 — GB/T 22239-2019

健身SaaS平台至少应满足**二级等保**；有健康数据则建议三级等保。

### 3.3 出口认证

| 目的地 | 要求 | 标准 |
|--------|------|------|
| 欧盟 | CE认证 | RED指令2014/53/EU（ETSI EN 300 328用于BLE） |
| 美国 | FCC认证 | Part 15B + Part 15C（BLE需FCC ID） |
| 中国国内 | SRRC型号核准 | BLE 2.4GHz需取得无线电发射设备型号核准 |

## 四、FHIR国际标准 — 与HealthHub的关联

- HL7 FHIR R4已被国家卫生健康委纳入交换标准
- FHIR Device + Observation资源直接对应GY-BLE25T的IMU数据
- 国家医保局平台已完成FHIR R4省级互操作性测试
- HealthHub数据层遵循FHIR标准，可直接对接国家全民健康信息平台
