## 1. 执行摘要

本文档定义C端微信小程序MVP产品方案，作为B2B2C规模化战略的关键触点。通过机构邀请码关联B端老客户，为终端用户提供个性化运动/睡眠干预服务，积累用户规模和数据飞轮。

**MVP目标**: 4周内小程序上线，触达首批种子用户，建立干预效果数据闭环。

**核心价值**: 干预效果可见、依从性可视化、每日个性化处方。

## 3. 数据模型

### 3.1 核心实体

```
User (用户)
├── openid: string (微信 openid)
├── nickname: string
├── avatar: string (URL)
├── phone: string (nullable)
├── invite_code: string (关联的机构邀请码)
├── institution_id: string (关联机构ID)
├── created_at: datetime
└── status: enum (active/inactive)

UserProfile (用户健康档案)
├── user_id: string
├── health_radar: object (5维度评分: 运动/睡眠/营养/压力/心肺)
├── target_goals: object (运动时长目标, 睡眠时长目标)
└── updated_at: datetime

DailyTask (每日任务)
├── id: string
├── user_id: string
├── date: date
├── task_type: enum (exercise/sleep)
├── prescription: object (运动处方 or 睡眠处方)
├── status: enum (pending/completed/skipped)
├── completed_at: datetime (nullable)
└── compliance_source: enum (manual/device_auto)

DeviceBinding (设备关联)
├── user_id: string
├── device_id: string
├── device_type: enum (wearable/scale/sleep_monitor)
├── bound_at: datetime
└── status: enum (active/inactive)

Achievement (成就)
├── id: string
├── user_id: string
├── badge_type: string
├── earned_at: datetime
└── display_order: int
```

### 3.2 微信小程序 Storage

| Key | 类型 | 说明 |
|-----|------|------|
| `userinfo` | object | 当前用户基本信息 (缓存) |
| `session_key` | string | 微信 session_key |
| `last_sync` | timestamp | 最后数据同步时间 |

## 4. 用户故事 (US-C系列)

### US-C-001: 首页健康看板

**优先级**: P0

**用户故事**:
> 作为终端用户，我需要一眼看到今日运动和睡眠数据及达标状态，以便快速了解自身健康状况。

**验收标准**:
- Given 用户打开小程序并进入首页
- When 页面加载时
- Then 显示今日运动卡片：步数、Active时长、达标状态（绿/黄/红）
- And 显示今日睡眠卡片：睡眠时长、质量评分、达标状态
- And 显示本周趋势小图（近7天折线图）

- Given 用户当日运动/睡眠已达标
- When 页面渲染时
- Then 对应卡片显示绿色"达标"标识

- Given 用户尚未关联设备
- When 页面渲染时
- Then 卡片显示"暂无数据，请关联设备"提示
- And 点击跳转设备关联页

### US-C-002: 每日干预任务执行

**优先级**: P0

**用户故事**:
> 作为终端用户，我需要清晰看到当日运动和睡眠干预任务，并能便捷标记完成或跳过，以便管理每日健康计划。

**验收标准**:
- Given 用户进入每日任务页
- When 页面加载时
- Then 显示当日任务列表：运动处方任务 + 睡眠处方任务

- Given 运动处方任务
- Then 显示：运动类型、推荐时长/强度、预计消耗卡路里
- And 提供"已完成"按钮和"跳过"选项

- Given 睡眠处方任务
- Then 显示：推荐睡眠时段、目标时长、睡眠小贴士
- And 提供"已完成"按钮和"跳过"选项

- Given 用户点击"已完成"
- When 点击时
- Then 任务状态更新为 completed
- And 首页卡片数据刷新
- And 依从率数据更新

- Given 用户点击"跳过"
- When 点击时
- Then 任务状态更新为 skipped
- And 记录跳过原因（可选）

### US-C-003: 进度追踪与成就

**优先级**: P0

**用户故事**:
> 作为终端用户，我需要看到自己的长期干预依从性趋势和成就徽章，以获得持续使用的动力和成就感。

**验收标准**:
- Given 用户进入进度追踪页
- When 页面加载时
- Then 显示本月依从率（实际完成任务/应完成任务）
- And 显示连续坚持天数
- And 显示近7天/30天趋势折线图

- Given 用户连续完成3天任务
- When 进入进度页时
- Then 显示"连续3天"小徽章

- Given 用户累计完成10天任务
- When 进入进度页时
- Then 显示"坚持10天"成就徽章

- Given 用户点击某个成就徽章
- When 点击时
- Then 显示徽章获得时间和对应任务记录

### US-C-004: 健康档案查看

**优先级**: P0

**用户故事**:
> 作为终端用户，我需要查看自己的健康档案，了解健康画像评分和关联的机构/设备信息。

**验收标准**:
- Given 用户进入"我的档案"页
- When 页面加载时
- Then 显示：头像、昵称、关联机构名称
- And 显示健康画像雷达图（5维度：运动/睡眠/营养/压力/心肺）

- Given 用户已关联设备
- Then 显示设备列表：设备名称、类型、绑定时间

- Given 用户由机构健康师管理
- Then 显示健康师姓名和联系方式（点击拨打电话）

- Given 用户尚未完成健康画像评估
- Then 显示"去评估"入口，引导完成画像创建

### US-C-005: 机构邀请码注册

**优先级**: P0

**用户故事**:
> 作为新用户，我需要通过机构提供的邀请码完成注册，并关联到对应机构，以便获取个性化干预服务。

**验收标准**:
- Given 用户首次打开小程序
- When 引导页时
- Then 提示输入机构邀请码（6位字母数字）

- Given 用户输入有效邀请码
- When 提交时
- Then 关联到对应机构
- And 跳转首页

- Given 用户输入无效邀请码
- When 提交时
- Then 显示"邀请码无效，请核对后重试"
- And 允许重新输入

- Given 用户没有邀请码
- When 选择"暂不绑定"时
- Then 创建个人账号（功能受限）
- And 显示提示"绑定机构可享受更多服务"

### US-C-006: 消息通知提醒

**优先级**: P1

**用户故事**:
> 作为终端用户，我需要收到每日的干预任务提醒，以便记得执行健康计划。

**验收标准**:
- Given 用户已开启通知权限
- When 每日早8点时
- Then 推送消息：今日任务提醒
- And 消息体包含：运动目标 + 睡眠目标

- Given 用户在设置页关闭通知
- When 系统推送时
- Then 不发送通知

## 5. 页面流程图

### 5.1 新用户首次打开流程

```
小程序启动
    ↓
引导页: 输入邀请码 或 暂不绑定
    ↓
微信授权 (获取 openid + 昵称头像)
    ↓
注册/登录
    ↓
首页 (健康看板)
```

### 5.2 每日使用流程

```
打开小程序
    ↓
首页Tab → 查看今日数据看板
    ↓
任务Tab → 查看/执行每日任务
    ↓         ↓              ↓
      完成      跳过         查看详情
         ↓         ↓
       更新依从率   记录跳过
             ↓
进度Tab → 查看依从趋势 + 成就
    ↓
我的Tab → 健康档案 / 设备关联 / 设置
```

### 5.3 设备数据同步流程

```
用户绑定设备 (蓝牙配对)
    ↓
设备数据上报 → 云端
    ↓
小程序首页拉取最新数据 (每5分钟自动刷新)
    ↓
手动下拉刷新也可触发
```

## 6. 技术约束

### 6.1 框架与平台

| 项目 | 约束 |
|------|------|
| 平台 | 微信小程序 (iOS + Android) |
| 框架 | 原生开发 (或 Taro/uni-app 多端) |
| 状态管理 | MobX / Redux (轻量) |
| UI 组件 | Vant Weapp / WeUI |
| 图表 | ECharts (小程序适配版) |

### 6.2 API 对接 (复用 intervention-engine)

| 端点 | 方法 | 用途 | 调用频率 |
|------|------|------|----------|
| `/api/users/{id}/profile` | GET | 获取用户健康画像 | 页面加载时 |
| `/api/users/{id}/metrics/today` | GET | 获取今日运动/睡眠数据 | 每5分钟 |
| `/api/users/{id}/tasks/daily` | GET | 获取当日任务列表 | 页面加载时 |
| `/api/users/{id}/tasks/{taskId}/complete` | POST | 标记任务完成 | 用户点击时 |
| `/api/users/{id}/tasks/{taskId}/skip` | POST | 跳过任务 | 用户点击时 |
| `/api/users/{id}/progress` | GET | 获取依从率趋势数据 | 页面加载时 |
| `/api/users/{id}/achievements` | GET | 获取成就列表 | 页面加载时 |
| `/api/institutions/{code}/verify` | POST | 验证邀请码 | 注册时 |

### 6.3 微信特有能力

| 能力 | 用途 |
|------|------|
| 微信 openid 登录 | 用户身份识别 |
| 微信支付 (暂不支持) | 未来付费场景 |
| 微信运动步数 (需用户授权) | 运动数据补充 |
| WebSocket | 设备数据实时推送 (可选) |
| 订阅消息 | 每日任务提醒 (需用户授权) |

### 6.4 性能约束

| 指标 | 要求 |
|------|------|
| 首屏加载 | < 3秒 (4G) |
| 页面切换 | < 300ms |
| 数据刷新 | < 1秒 |
| 小程序包体积 | < 2MB |

## 7. 数据安全与合规

- 用户健康数据存储需符合《个人信息保护法》
- 隐私政策需在首次启动时展示并获同意
- 机构邀请码仅用于关联，不存储敏感信息
- 设备数据采用 HTTPS 传输
