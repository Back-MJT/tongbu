## 1. 概述

### 1.1 文档目的

本文档定义 HealthHub 第零层（IMU+BLE 硬件模组）的固件架构，目标是为宁津县传统抗阻器械提供数字化能力。固件运行在 Nordic nRF52840 芯片上，基于 Zephyr RTOS，驱动 ST LSM6DSO IMU 传感器，通过 BLE 5.0 广播组数/使用数据。

### 1.2 核心约束

| 约束项 | 值 |
|--------|-----|
| BOM 成本 | ¥25-45 / 个（千片量） |
| 采样频率 | 50 Hz |
| BLE 广播 | 非连接模式（Eddystone-UID 或自定义） |
| 供电 | 框架取电（优先）/ CR2032 备选 |
| 防护等级 | IP54 |
| 安装方式 | 热缩螺丝固定，3 分钟完成 |
| 典型器械 | 哑铃架、史密斯机、综合训练架、绳索机 |

### 1.3 硬件选型依据

| 组件 | 选型 | 理由 |
|------|------|------|
| 主控 | Nordic nRF52840 | BLE 5.0 内置（Cortex-M4），国内方案成熟，Zephyr 生态完善 |
| IMU | ST LSM6DSO（6轴） | 工业级稳定性，低功耗（2.5mA），IP54 封装成熟 |
| 电源管理 | AXP2101 / 简易 LDO | 框架取电时需要 DC-DC 降压；CR2032 备选直接供电 |

## 3. Zephyr RTOS 架构

### 3.1 为什么用 Zephyr

1. **BLE 栈完善**：原生支持 BLE 5.0 Host 协议栈，无需移植
2. **功耗管理**：提供 Tickless Kernel、Device Runtime PM、System OFF 模式
3. **驱动生态**：LSM6DSO 已有社区驱动（STSensorStack）
4. **构建工具**：west + Devicetree，跨芯片平台统一
5. **License**：Apache 2.0，商业友好

### 3.2 线程设计

```
main()                    ← 系统初始化，启动各线程
  └── k_thread_create(&imu_thread)      [优先级：高]
  │       消费线程：消费 IMU 原始 buffer
  │       触发 rep_counter 计数逻辑
  │
  └── k_thread_create(&bleadv_thread)    [优先级：低]
          广播线程：定时（1Hz）从状态区读取数据，填充 BLE 广播 payload
```

| 线程 | 优先级 | 栈大小 | 周期 | 职责 |
|------|--------|--------|------|------|
| `imu_thread` | 8 (高) | 512 B | 20ms（50Hz） | 读取 IMU，触发计数算法 |
| `bleadv_thread` | 10 (低) | 384 B | 1000ms | BLE 广播发送 |
| `idle_thread` | 15 (系统) | 256 B | — | System OFF 息眠 |

## 4. LSM6DSO 驱动

### 4.1 寄存器配置（50Hz 采样）

| 寄存器 | 值 | 说明 |
|--------|---|------|
| `CTRL1_XL` (0x10) | `0x3C` | ±4g，50Hz ODR，低功耗 |
| `CTRL2_G` (0x11) | `0x30` | ±250°/s，50Hz ODR |
| `CTRL3_C` (0x12) | `0x44` | 使能 DRDY，非连续写 |
| `INT1_CTRL` (0x0E) | `0x03` | DRDY 映射到 INT1 |

### 4.2 驱动代码结构

```c
/* 位置：boards/arm/nrf52840_dongle/lsm6dso.c */
#include <zephyr/drivers/sensor.h>
#include <zephyr/device.h>

#define LSM6DSO_I2C_ADDR 0x6B

static const struct device *lsm6dso_dev;

/* 初始化 */
int lsm6dso_init(void) {
    lsm6dso_dev = DEVICE_DT_GET_ANY(st_lsm6dso);
    if (!device_is_ready(lsm6dso_dev)) {
        return -ENODEV;
    }

    /* 配置 50Hz ±4g */
    struct sensor_config cfg = {
        .odr = 50,
        .fs = 4,
    };
    sensor_attr_set(lsm6dso_dev, SENSOR_CHAN_ACCEL_XYZ, 
                    SENSOR_ATTR_CONFIGURATION, &cfg);
    return 0;
}

/* 读取加速度（合加速度）*/
float lsm6dso_read_magnitude(void) {
    struct sensor_value accel[3];
    sensor_sample_fetch(lsm6dso_dev);
    sensor_channel_get(lsm6dso_dev, SENSOR_CHAN_ACCEL_XYZ, accel);

    float ax = sensor_value_to_double(&accel[0]);
    float ay = sensor_value_to_double(&accel[1]);
    float az = sensor_value_to_double(&accel[2]);

    /* 合加速度 = sqrt(ax² + ay² + az²)，减去重力分量后 */
    return sqrtf(ax*ax + ay*ay + az*az);
}
```

## 5. 组数计数算法

### 5.1 算法原理

基于 **振动峰值检测 + 静默期分组**：
- 采样频率 50Hz，合加速度经高通滤波（去掉重力分量）
- 峰值超过阈值 1.5g 计为一次 rep
- 超过 500ms 的峰值才计入（过滤抖动）
- 超过 30 秒静默判定为本组结束

### 5.2 伪代码 → C 实现

```c
/* lsm6dso.c — 组数计数核心算法 */

#define REPS_THRESHOLD_G      1.5f     /* g */
#define REPS_DEBOUNCE_MS      500      /* ms，两次 rep 最小间隔 */
#define SET_REST_TIMEOUT_MS   30000    /* ms，超过此时间判定组结束 */
#define SAMPLE_FREQ_HZ        50
#define SAMPLE_PERIOD_MS       (1000 / SAMPLE_FREQ_HZ)  /* 20ms */

typedef struct {
    int rep_count;          /* 当前组计数 */
    int set_count;          /* 历史组数（累计） */
    uint32_t last_rep_time; /* 上次 rep 时间戳（系统运行时间，ms） */
    uint32_t set_start_time;/* 当前组开始时间 */
    uint32_t session_start;  /* 本次使用开始时间 */
    bool in_session;        /* 是否在使用中 */
} rep_counter_state_t;

static rep_counter_state_t state = {0};
static float history_buffer[SAMPLE_FREQ_HZ * 2]; /* 2秒滑动窗口 */
static int buf_idx = 0;

/* 高通滤波（单极 RC，fc=3Hz） */
float highpass_filter(float x) {
    static float x_prev = 0;
    static float y_prev = 0;
    float alpha = 0.114; /* fc=3Hz @ 50Hz */
    float y = alpha * (y_prev + x - x_prev);
    x_prev = x;
    y_prev = y;
    return y;
}

/* 计算合加速度（已去除重力） */
float compute_magnitude(void) {
    float ax = lsm6dso_read_axis(ACCEL_X);
    float ay = lsm6dso_read_axis(ACCEL_Y);
    float az = lsm6dso_read_axis(ACCEL_Z) - 1.0f; /* 去掉1g重力 */
    return sqrtf(ax*ax + ay*ay + az*az);
}

/* 峰值检测（带防抖）*/
bool detect_rep(uint32_t now_ms) {
    float mag = fabs(compute_magnitude());

    /* 高通滤波去重力漂移 */
    mag = highpass_filter(mag);

    /* 阈值判断 */
    if (mag > REPS_THRESHOLD_G) {
        /* 防抖：距上次 rep 必须超过 500ms */
        if (now_ms - state.last_rep_time > REPS_DEBOUNCE_MS) {
            state.last_rep_time = now_ms;
            return true;
        }
    }
    return false;
}

/* 组结束判断和上报 */
void check_set_end(uint32_t now_ms) {
    if (state.in_session && (now_ms - state.last_rep_time > SET_REST_TIMEOUT_MS)) {
        /* 组结束，上报云端 */
        uint32_t duration_ms = now_ms - state.set_start_time;
        float intensity = (float)state.rep_count / (duration_ms / 60000.0f); /* reps/min */

        bleadv_emit_set_data(state.rep_count, duration_ms, intensity);
        state.set_count++;
        state.rep_count = 0;
        state.in_session = false;
    }
}

/* IMU 消费线程（50Hz）*/
void imu_thread_entry(void *arg1, void *arg2, void *arg3) {
    uint32_t last_wake_time = k_uptime_get_32();

    while (1) {
        uint32_t now_ms = k_uptime_get_32();

        /* 读取并计数 */
        if (detect_rep(now_ms)) {
            state.rep_count++;
            state.in_session = true;
            if (state.set_start_time == 0) {
                state.set_start_time = now_ms;
            }
        }

        /* 组结束检查 */
        check_set_end(now_ms);

        /* 50Hz 周期睡眠 */
        last_wake_time = k_thread_timeout_set(&k_current_thread(),
            K_MSEC(SAMPLE_PERIOD_MS));
    }
}

/* 获取当前状态（BLE 广播线程读取）*/
rep_counter_state_t rep_counter_get_state(void) {
    return state;
}
```

### 5.3 精度保障

| 场景 | 预期误差 | 原因 |
|------|---------|------|
| 标准动作（哑铃弯举、卧推） | <5% | 加速度峰值明显 |
| 慢速离心动作 | 5-15% | 峰值不明显，需调低阈值 |
| 抖动/晃动（误触发） | <1% | 500ms 防抖过滤 |
| 多人共用设备区分 | 不支持 | 依赖上层（手机/网关）做时间窗口推断 |


## 6. BLE 广播数据格式设计

### 6.1 广播策略

- **模式**：非连接广播（Broadcaster），节省电量
- **UUID**：使用 Eddystone-UID（Google）或自定义 128-bit UUID
- **广播频率**：1Hz（平衡功耗与实时性）
- **有效负载**：最多 31 bytes（BLE 4.x 广播包限制）

### 6.2 自定义 Service Data 格式

```
字段：
  [0-1]   Flags (0x02 0x01) = LE General Discoverable
  [2-3]   Service UUID (0x03 0x18) = Health Device Profile（复用）
  [4-27]  Service Data（26 bytes）：

  Byte 0:     Protocol Version (0x01)
  Byte 1:     Device Type Flags (0b00000001)  bit0=IMU, bit1=HasPower
  Byte 2-3:   Device ID（16-bit，模组唯一ID，从 flash 读取）
  Byte 4-5:   Firmware Version (BCD, e.g. 0x0100 = v1.0.0)
  Byte 6:     Battery Level (%) — 0xFF=外部供电
  Byte 7:     Current Rep Count (本组)
  Byte 8:     Current Set Count (累计，本 session)
  Byte 9-10:  Session Duration (秒)
  Byte 11-12: Last Rep Magnitude (0.01g resolution)
  Byte 13-14: Rest Time (秒，距上次 rep)
  Byte 15:    RSSI @ 1m (signed, dBm)
  Byte 16-27: Reserved (checksum, future use)

  [28-30]  Eddystone-UID namespace + instance（可选，用于多设备定位）
```

### 6.3 BLE 配置命令（手机→模组，写入特征值）

| Command ID | 功能 | Payload |
|------------|------|---------|
| `0x01` | 设置设备 ID | [DeviceID_L, DeviceID_H] |
| `0x02` | 设置阈值 | [threshold_0.1g] |
| `0x03` | 重置计数器 | (none) |
| `0x04` | 进入 OTA 模式 | (none) |
| `0x05` | 读取 MAC 地址 | 返回模组 BLE MAC |

### 6.4 OTA 升级

使用 Nordic Secure DFU（通过 BLE），固件 binary 分为：
- **SoftDevice**（BLE 协议栈，固定不变）
- **Application**（用户应用，可 OTA 更新）
- **Bootloader**（nRF5 SDK Bootloader）

OTA 最大包大小：244 bytes，预期升级时间约 30-60 秒。
