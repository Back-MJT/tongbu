## 背景说明

Board指令 (XIN-98 §1)：产品的底层逻辑是相通的，只是应用的环境不同，方法论不同。HealthHub的核心能力——IMU动作感知 + 数据上云 + Claude API训练引擎——在三个核心场景中形态不同，但本质相同：**把身体运动数字化，然后给出个性化反馈**。

本文档定义三个核心场景的典型用户画像，每个场景2个典型persona，共6个画像。服务于产品设计时的同理心锚定和功能优先级判断。

## 场景一：健身房 — 年轻力量用户（20-35岁）

### Persona A：陈浩宇，27岁，互联网产品经理

**人口统计**
- 年龄：27岁
- 职业：互联网公司产品经理
- 城市：上海
- 月收入：¥18,000-25,000
- 居住：合租公寓，距公司地铁30分钟
- 婚恋状态：单身，忙于工作

**生活状态**
- 工作日平均下班时间22:00以后，健身时间碎片化
- 周末相对规律，有完整2-3小时训练块
- 健身习惯2年，主要在公司附近健身房撸铁
- 熟悉Keep、Fiture等主流健身App，有一定健身知识

**运动习惯**
- 每周3-4次力量训练（胸背腿循环）
- 有氧穿插：偶尔跑步或椭圆机
- 训练目标：增肌、保持体型、提高体能
- 训练痛点：不知道动作对不对、发力感差、进步缓慢

**核心需求**

| 类型 | 需求 |
|------|------|
| 功能 | 动作质量实时反馈（我做了10组，但有几组是"有效"的？） |
| 功能 | 训练记录自动积累，不用手动记 |
| 功能 | 知道今天练哪里、练多少，不用自己想计划 |
| 情感 | 有人"看见"我的努力，而不是数据冷冰冰 |
| 情感 | 进步可视化，有成就感 |

**产品使用场景**
- 手机扫器械二维码，绑定当前训练动作
- 训练中：手表/手机震动提醒，提示"离心控制再慢一点"
- 训练后：微信小程序推送训练报告，告知本次有效组数/动作质量评分
- 周/月：查看进步曲线

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 不知道动作做对没有，看视频学但实操还是错 | 实时告诉我哪里不对 |
| 训练计划凭感觉，这次练多少全看心情 | 给我一个合理的计划 |
| 练完不记得上次练了什么、用了什么重量 | 自动记录，历史可查 |
| 办了卡但进步慢，动力消退 | 看到进步曲线，动力维持 |

**产品设计启示**
- **即时反馈优先**：训练中的实时提示比训练后的报告更有价值（行为改变窗口在运动中）
- **数据要"有意义"**：不说"峰值角速度0.23rad/s"，说"你这组比上次深蹲多停留了0.5秒，刺激更深"
- **社交价值**：可分享训练成果到朋友圈，满足炫耀需求
- **省时间**：用户核心时间是稀缺资源，交互要快，不超过3秒

### Persona B：林晓晨，23岁，应届毕业生，健身新手

**人口统计**
- 年龄：23岁
- 职业：刚入职财务助理
- 城市：杭州
- 月收入：¥6,000-8,000
- 居住：父母家，通勤30分钟
- 婚恋状态：单身，下班后时间相对充裕

**生活状态**
- 入职3个月，第一次正式进健身房
- 健身知识薄弱，买了网课但动作还是看不懂
- 容易害羞，不敢问教练，也不敢问旁边的人
- 目标是减脂+增肌，希望半年内有明显改变

**运动习惯**
- 每周2-3次，以跑步机热身+跟着App练动作为主
- 不敢碰自由重量，怕动作丑被人笑
- 训练强度偏低，经常练完没什么感觉
- 还在建立运动习惯，间歇性勤奋

**核心需求**

| 类型 | 需求 |
|------|------|
| 功能 | 告诉我每个动作怎么做、做得对不对 |
| 功能 | 没有教练在场的情况下也能安全训练 |
| 功能 | 训练不太累、不太难，容易坚持 |
| 情感 | 不要被人笑话，不要尴尬 |
| 情感 | 有人鼓励，而不是评判 |

**产品使用场景**
- 进健身房先用手机扫器械，小程序弹出动作示范
- 跟着屏幕上的动作指引练习，做错了有温和提示而不是批评
- 训练结束收到"今天表现不错！"的鼓励
- 看到其他新手用户的进步，给自己打气

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 不知道怎么练，怕受伤 | 给我一个安全、简单的开始 |
| 没人教，看视频还是学不会 | 有个"虚拟教练"随时在身边 |
| 练完没感觉，觉得浪费时间 | 让我真的感受到锻炼的效果 |
| 坚持不下来，练几次就不想去了 | 建立信心，让我看到自己的进步 |

**产品设计启示**
- **新手友好**：首次连接时，3步之内开始第一个动作，不过度解释原理
- **正向反馈优先**：做对了要肯定，做错了要引导，不要负面措辞
- **安全感**：强调"安全"和"正确"而不是"强度"或"极限"
- **虚拟陪伴感**：晓晨需要感觉到"有人在"，哪怕是AI

## 场景二：老年康复馆 — 刚退休活力老人（55-70岁）

### Persona C：赵叔，62岁，刚退休干部

**人口统计**
- 年龄：62岁
- 职业：去年刚退休（某县级机关干部）
- 城市：德州（宁津附近）
- 月收入：退休金¥5,000-7,000，有一定积蓄
- 居住：自购房，与老伴同住
- 婚恋状态：已婚，子女在外地工作

**生活状态**
- 刚退休1年，从"每天有事做"突然变成"天天在家"
- 身体状况总体良好，但有轻度高血压、高血脂
- 闲不住，不喜欢广场舞（"太吵"）和棋牌室（"没意思"）
- 内心有"不服老"的冲动，想证明自己还能有所追求
- 开始关注健康，但不知道从哪里系统开始

**Board洞察（XIN-98 §3）**：刚退休的人，既对健康有需求，又可以释放那颗束缚的心，终于可以开始追求人生的感觉。

**运动习惯**
- 以前几乎不运动，现在开始每天走路
- 偶尔骑行，但不知道练什么、怎么练
- 目标：不得病、有精神、给子女减少负担
- 愿意为健康花钱，但要有实际效果

**核心需求**

| 类型 | 需求 |
|------|------|
| 功能 | 告诉我练什么、怎么练、练多少 |
| 功能 | 运动安全，不受伤 |
| 功能 | 能看到健康指标的变化（血压、体重） |
| 情感 | 被当作一个"还有追求的人"对待，而不是老人 |
| 情感 | 有社群，有同伴，不孤独 |
| 情感 | 被理解那种"退休后的空虚感" |

**产品使用场景**
- 康复馆里使用上肢功率车或椭圆机，手腕戴着轻量传感器
- 设备屏幕显示"赵叔，今天我们练15分钟，强度适中"
- 训练中有语音提示"很好，呼吸很稳"
- 训练后，设备显示本次运动心率和教练点评，推送到手机
- 康复馆内有排行榜，"赵叔本周运动时长排第3"

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 不知道怎么科学运动，想练但怕练坏 | 有一个专属的、专业的运动计划 |
| 一个人练很无聊，难以坚持 | 有同伴，有社群，有互动 |
| 练了一段时间不知道有没有效果 | 看到指标改善，有成就感 |
| 子女不在身边，有孤独感 | 通过设备连接外界，有存在感 |

**产品设计启示**
- **语言要尊重**：称呼"赵叔"而不是"老年人"，说"力量训练"不说"老人康复"
- **大字体、高对比度**：视力下降，屏幕要清楚
- **社群连接**：排行榜、活动、社交分享，让老人感到被关注
- **成就系统**：集邮式成就，让运动变得有"收集"乐趣
- **子女关联**：数据可分享给子女，子女能看到父母在运动，有安心感和督促感

### Persona D：周阿姨，58岁，康复馆活跃会员

**人口统计**
- 年龄：58岁
- 职业：全职太太，之前从未有正式工作
- 城市：德州
- 月收入：配偶退休金支撑，家庭经济压力小
- 居住：与配偶和80岁婆婆同住
- 婚恋状态：已婚，照顾家庭为主

**生活状态**
- 婆婆需要照顾，日常围着家务转
- 个人时间碎片化，运动时间依赖家庭安排
- 重视健康，但自己的需求往往排在最后
- 在康复馆认识了一些同龄朋友，社交意义大于运动本身
- 性格开朗，喜欢聊天，关心同伴的进步

**运动习惯**
- 每周3-5次去康复馆（与朋友结伴）
- 喜欢集体活动：广场舞、太极、康复器械集体课
- 强度偏低，以低冲击有氧和拉伸为主
- 运动目标：保持关节灵活、不给子女添麻烦、有事做不无聊

**核心需求**

| 类型 | 需求 |
|------|------|
| 功能 | 简单、好操作，不要复杂 |
| 功能 | 和朋友一起练，有互动 |
| 功能 | 身体状况能被记录，给子女看 |
| 情感 | 运动是生活乐趣，不是任务 |
| 情感 | 和朋友比肩的感觉，有人陪伴 |
| 情感 | 被肯定、被看见 |

**产品使用场景**
- 到康复馆，扫二维码，和闺蜜一起开始训练
- 设备显示"周阿姨加油，您今天又来了！"
- 训练中有提示，但节奏慢，允许暂停休息
- 结束后和闺蜜的"运动成绩"对比，互相打趣
- 康复馆工作人员推送"周阿姨本周出勤5天，获得全勤徽章"

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 设备操作复杂，搞不懂 | 扫码即用，不用学 |
| 运动太无聊，难以坚持 | 有朋友、有互动、有乐趣 |
| 感觉不被重视，只是来凑热闹 | 让我感觉到我的参与有价值 |
| 不知道练得对不对，没人指导 | 有教练/AI在关注我 |

**产品设计启示**
- **极简交互**：扫一扫就开始，不要登录、不要设置
- **社交设计**：训练数据可对比，排行榜支持好友互动
- **趣味性**：徽章、连续打卡、成就收集，让运动游戏化
- **语音提示为主**：阿姨们更习惯听而不是看屏幕
- **允许"低强度"**：不是每个人都要练到气喘吁吁，"来了就好"也是价值

## 场景三：家庭场景 — 中青年家庭用户（25-50岁）

### Persona E：王建国，38岁，双职工爸爸

**人口统计**
- 年龄：38岁
- 职业：制造业中层管理
- 城市：宁津（德州）
- 月收入：¥10,000-15,000
- 居住：自购房，与配偶和两个孩子（10岁、7岁）同住
- 婚恋状态：已婚，配偶也在工作

**生活状态**
- 典型"夹心层"：上面有老人，下面有孩子，中间是自己
- 工作忙，应酬多，运动时间极度碎片化
- 知道运动重要，但总是"等有时间再说"
- 家里有跑步机，但沦为晾衣架
- 健康状况：体检轻度脂肪肝，尿酸偏高

**运动习惯**
- 办了健身房卡但很少去（平均每月1-2次）
- 家里有设备但不知道怎么用
- 目标：减脂、改善脂肪肝、有更多精力陪孩子
- 愿意为家庭健康付费，但要求省心

**核心需求**

| 类型 | 需求 |
|------|------|
| 功能 | 碎片化时间能用，10分钟也能练 |
| 功能 | 不需要想训练计划，设备告诉我怎么练 |
| 功能 | 运动数据积累，能看到健康趋势 |
| 功能 | 孩子的运动数据也能一起看 |
| 情感 | 不需要多专业，不要让我觉得自己懒 |
| 情感 | 家庭共同的健康目标，有责任感 |

**产品使用场景**
- 晚上22:00，等孩子睡了，在家用划船机上练10分钟
- 手机碰一下设备，自动开始训练
- 训练中偶尔看一下屏幕，知道自己做得怎么样
- 训练后，数据自动同步到家庭健康面板
- 周末和孩子一起用设备，做简单的亲子运动

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 没时间，一次训练要1小时太奢侈 | 10分钟也能有效 |
| 不知道练什么、怎么练 | 设备告诉我每天练什么 |
| 一个人练很难坚持 | 有家庭目标，有家庭成员互相促进 |
| 设备买回来落灰 | 让设备成为家庭日常的一部分 |

**产品设计启示**
- **时间弹性**：支持5分钟、10分钟、30分钟多种训练块
- **自动化优先**：碰一碰就连接，不需要手动选择动作
- **家庭视图**：一个App看到全家人的运动数据
- **低门槛内容**：不需要学习，对着练就行，不要复杂术语
- **提醒而非催促**：定时提醒，但不过度打扰

- 职业：全职妈妈（孩子5岁，全天照顾）
- 城市：济南

- 居住：自购房，与配偶和女儿同住
- 婚恋状态：已婚，全职照顾家庭

- 孩子上幼儿园后，白天有3-4小时可支配时间
- 对自己外貌/健康有追求，但总把家庭放在前面

- 以前有健身习惯，现在中断3年
- 想恢复运动，但带孩子的情况下很难规律
- 喜欢在家练（节省时间，不用出门）
- 目标：恢复体型、增加精力、改善产后体质

| 类型 | 需求 |
|------|------|
| 功能 | 可以在家练，不用出门 |
| 功能 | 时间灵活，随时开始，不需准备 |
| 功能 | 动作简单，不需要专业基础 |
| 功能 | 能看到自己的进步，有成就感 |
| 情感 | 运动是给自己的时间，不是又一个义务 |
| 情感 | 有人在鼓励，不孤单 |
| 情感 | 变好的感觉，为自己做点什么 |

**产品使用场景**
- 孩子午睡时，用手机连接家用健身设备
- 跟着屏幕练15分钟，动作不复杂
- 训练后可分享"打卡"到妈妈群，获得社交肯定
- 看到自己的运动数据在变好，给自己打气
- 有时和群里认识的妈妈约一起视频练

**痛点与期望**

| 痛点 | 期望 |
|------|------|
| 时间碎片化，没法规划30分钟训练 | 10分钟也能练，随时开始 |
| 动作太难，跟不上 | 简单、有示范、容许跟不上 |
| 一个人练无聊，难以坚持 | 有社群，有互动，有人陪 |
| 看不到进步就想放弃 | 最小可见的进步，给继续的动力 |
| 老公觉得"在家闲着"，不理解 | 运动被看见、被认可 |

**产品设计启示**
- **极度灵活**：随时可停，容许被打断，不强制完整流程
- **社群功能**：妈妈们互相鼓励，打卡分享是核心动力
- **进步可视化**：体态变化、力量变化、精神状态变化，多维度认可
- **正向语言**：不说"你偷懒了"，说"今天已经迈出了一步"
- **家庭地位**：在家庭健康中的角色被认可，是"健康管理者"而非"消费者"

## 跨场景设计原则

从6个画像中提炼出HealthHub产品设计的通用原则：

### 1. 动作反馈要及时
陈浩宇（健身房）和赵叔（康复馆）都需要知道"做得对不对"，但时机不同：年轻人要实时中断提醒，老年人要结束后温和总结。**一个尺寸不适用所有场景。**

### 2. 数据要说"人话"
所有人都讨厌Raw Data。陈浩宇要看"有效组数"，陈婷要看"今天的你比昨天好"。**把数据翻译成用户能立即理解的意义。**

### 3. 情感需求和产品功能一样重要
- 陈浩宇需要"被看见" → 成就系统
- 赵叔需要"不孤独" → 社群连接
- 陈婷需要"属于自己的时间" → 运动记录和自我肯定

**产品设计不能只解决功能问题，要解决情感问题。**

### 4. 老年人需要特别对待
- 视力：字体要大，对比度要高
- 认知：步骤要少，提示要重复
- 语言：尊重而非俯就
- 安全：第一位的，优先于所有其他需求

### 5. 家庭场景是长期留存的关键
一个人可能放弃，但一家人互相促进更难放弃。王建国和陈婷的故事说明：**家庭视图、社群分享、家庭目标**是防止流失的核心机制。
