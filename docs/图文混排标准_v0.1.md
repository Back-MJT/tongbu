# 图文混排标准 v0.1

> 版本：v0.1  
> 状态：初稿，待评审  
> 发布日期：2026-04-24  
> 适用渠道：微信公众号 / 小红书 / 官网博客  

---

## 1. 目的

确保昕动智能对外发布的每一篇内容：
- **结构清晰**：读者能在 30 秒内判断内容是否相关
- **信息密度高**：图文互补，避免纯文字堆砌
- **品牌一致**：所有渠道使用统一的排版语言

---

## 2. 必选结构（五段式）

每篇长文（>500 字）必须包含以下五个模块：

### 2.1 元信息栏（文件头部）

```
> 适用渠道：微信公众号
> 目标受众：健身器械厂家老板 / 销售负责人
> 状态：首发 / 修订
> 创建时间：2026-04-15
> 配图：MiniMax image-01 生成
```

### 2.2 引言区

- 直接引用用户原话（提问）或行业数据
- 用 `>` 块引用格式
- 长度：50–100 字
- 目的：让读者产生共鸣，"这就是我想的"

### 2.3 核心配图

- 每个内容模块配备 **一张核心配图**
- 配图说明格式：`![配图标题](./filename.jpg)`
- 配图类型优先级：
  1. 数据可视化（图表、对比图）
  2. 示意图（流程图、架构图）
  3. 场景图（产品使用、展会现场）
  4. 信息图（一图读懂系列）

### 2.4 正文模块

- 使用 **二级标题**（`## 【模块名称】`）划分章节
- 每个章节：
  - 不超过 300 字
  - 包含至少一个要点列表（`- 要点描述`）
  - 要点需简洁、可操作
- 避免长段落（>5 行需拆分）

### 2.5 总结 / CTA

- 50–100 字
- 明确告诉读者下一步动作（如：扫码联系、下载白皮书）

---

## 3. 标题层级规范

```
# 大标题（文章主题）
## 【模块一】标题
### 小节标题（如有）
```

- H1 只出现一次（文章标题）
- H2 按模块划分，每篇不超过 5 个
- H3 仅在长文中使用（避免层级过深）

---

## 4. 图片规范

| 类型 | 格式 | 命名规范 | 存放位置 |
|------|------|----------|----------|
| 封面图 | JPG / PNG | `{E/H/I}_{序号}_{描述}.jpg` | 同目录 |
| 文中配图 | JPG / PNG | `{E/H/I}_{序号}_{描述}.jpg` | 同目录 |
| 数据图表 | PNG | `{E/H/I}_{序号}_chart_{描述}.png` | 同目录 |
| 示意图 | PNG / SVG | `{E/H/I}_{序号}_diagram_{描述}.png` | 同目录 |

**配图占位符**（初稿阶段）：
```
![配图标题](./placeholder.jpg)  <!-- TODO: 配图描述，生成后替换 -->
```
