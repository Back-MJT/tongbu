```typescript
  deviceId: string;
  metrics: {
    exerciseType?: string;     // 动作类型 (推/拉/蹲等)
    exerciseReps?: number;    // 重复次数
    load?: number;             // 负荷 (kg)
    power?: number;            // 即时功率 (W)
    movementVelocity?: number; // 动作速度 (m/s)
    exerciseDuration?: number; // 运动时长 (秒)
    exerciseIntensity?: number; // 运动强度 (1-10)
    caloriesBurned?: number;   // 消耗卡路里
    heartRate?: number;        // 心率 (bpm)
  };
}
```

### 2.2 IMU 传感器选型建议

> IMU (Inertial Measurement Unit) 用于采集运动时的加速度、角速度、姿态等数据，为力量训练动作分析和运动处方提供数据基础。

#### 2.2.1 选型核心参数

| 参数 | 最低要求 | 推荐规格 | 说明 |
|------|---------|---------|------|
| 采样率 | 50 Hz | **100-200 Hz** | 力量训练动作分析需捕捉快速发力 |
| 加速度量程 | ±16g | ±16g 或 ±32g | 力量训练冲击较大 |
| 陀螺仪量程 | ±2000°/s | ±2000°/s | 捕捉快速旋转动作 |
| 通信接口 | BLE 4.2+ | **BLE 5.0** | 低功耗蓝牙更适电池供电 |
| 功耗 | < 20mA | **< 10mA (BLE待机)** | 续航需支持连续采集 8h+ |
| 尺寸 | < 40x40x10mm | — | 便于绑缚在设备/肢体上 |
| 防水等级 | IP54 | **IP67** | 室内运动场景基本需求 |
| 成本 (单件) | < ¥200 | **¥100-¥200** | 试点阶段成本控制 |

#### 2.2.2 推荐 IMU 方案

| 方案 | 芯片 | 特点 | 参考成本 | 推荐场景 |
|------|------|------|---------|---------|
| 低成本入门级 | MPU-6050 (Invensense) | 6轴 (Acc+Gyro)，I2C/SPI | ¥15-¥30 | PoC验证 |
| **中端均衡型 (试点推荐)** | **MPU-6500 / BMI160** | 6轴，低功耗，小尺寸 | **¥30-¥60** | **试点首选** |
| 高精度型 | BMI088 | 6轴，高精度，抗冲击 | ¥80-¥150 | 高端场景 |
| 多传感器融合 | ICM-20948 (9轴) | 9轴，含磁力计，BLE | ¥80-¥120 | 研发阶段 |
| BLE一体化 | LilyGO T-Beam / 自组装 | BLE SoC + IMU 一体化 | ¥50-¥100 | 快速原型 |

**试点建议**: 采用 **MPU-6500 / BMI160** 方案 + 外接 BLE5.0 模块（如 NRF52840），成本与性能平衡最佳

#### 2.2.3 IMU 传感器安装方案

| 安装位置 | 适用场景 | 数据价值 | 实施难度 | 试点推荐 |
|---------|---------|---------|---------|---------|
| 固定在器械上 | 坐姿推胸、腿举等固定轨迹 | 中 | 低 | ✅ |
| 绑缚于用户手腕/手臂 | 自由重量训练 (哑铃、杠铃) | 高 | 中 | ✅ |
| 绑缚于用户腰部 | 深蹲、硬拉等髋部主导动作 | 高 | 中 | ✅ |
| 多传感器组合 | 综合评估 | 最高 | 高 | 后续扩展 |

**试点方案**: 器械固定 + 用户手腕双传感器组合，覆盖主要力量训练场景

### 2.3 IMU 数据到运动数据的转换算法需求

#### 2.3.1 原始数据规格

```
数据来源: IMU (加速度三轴 + 角速度三轴)
采样率: 100 Hz (推荐)
数据格式: 16位有符号整数，量程±16g / ±2000°/s
通信协议: BLE characteristic 通知 / WiFi UDP
```

#### 2.3.2 运动数据转换算法需求

| 算法模块 | 输入 | 输出 | 技术要求 |
|---------|------|------|---------|
| 姿态估计 | 加速度+角速度原始数据 | 欧拉角/四元数姿态 | 100Hz 实时，误差<5° |
| 动作分段 | 连续姿态序列 | 动作开始/结束时刻 | 识别RM（重复最大次数） |
| 力量训练动作分类 | 动作分段 + 时序特征 | 动作类型 (推/拉/蹲/拉等) | 准确率>85% (需训练数据) |
| 运动强度评估 | 动作速度 + 负荷 | RM估算、组间休息建议 | 需与设备联动或用户输入负荷 |
| 运动处方生成 | 用户画像 + 动作数据 | 个性化运动处方 | 规则引擎v0.1驱动 |

#### 2.3.3 技术可行性依赖

- **XIN-36 (IMU传感器集成可行性评估)** — 进行中，由工程团队负责
- 关键依赖: IMU数据质量标定方案、多传感器时间同步方案

## 五、通用数据接入层 — 统一 Device Schema

> 见 `DEVICE_SCHEMA_UNIFIED.md` — 所有客户设备数据统一映射至此 schema，消除具体协议差异。

**设计原则**:
1. **协议无关**: 不论 BLE/MQTT/HTTP/私有协议，进来后统一为 DeviceData
2. **设备无关**: 支持力量训练、心率带、睡眠监测、体脂秤等全品类设备
3. **可扩展**: `data` 字段为 Record<string, any>，支持任意设备特定字段
4. **时序优先**: 统一使用毫秒时间戳，便于时序数据库处理
