# 昕动智能团队建设方案

> 作者: CEO Agent
> 日期: 2026-04-08
> 状态: 待董事会确认

## 1. 现状

- 公司: 宁津昕动智能科技有限公司
- 目标: 18个月内达成1亿营收，从200万启动资金起步
- 核心方向: 健康装备数字底座 + 个体化干预引擎
- 现有agent: CEO + Founding Engineer

## 2. Phase 1 (0-12月) 团队规划

### P0: 必须立即到岗

**产品经理 (Product Manager)**
- 能力要求: 用户研究、需求分析、竞品对标、产品路线图规划、PRD撰写
- 关键产出:
  - 数字底座MVP产品定义文档
  - 干预引擎v0.1产品规格
  - 首批验证场景的产品方案
  - B2B客户需求调研报告
- 到岗后第一件事: 走访2-3家目标机构(健康管理机构/康复医院)，产出需求洞察

**市场/BD经理 (Business Development)**
- 能力要求: B2B销售策略、机构客户开拓、合作伙伴管理、商务谈判
- 关键产出:
  - B2B客户开拓计划和目标机构清单
  - 首批2-3家合作机构签约
  - 宁津本地产业资源对接方案
  - 竞争格局分析报告
- 到岗后第一件事: 建立宁津本地健康产业网络，锁定首批2-3家合作机构

### P1: Phase 1中期到岗

**数据科学家 (Data Scientist)**
- 能力要求: 健康数据建模、机器学习、时序分析、A/B测试设计
- 到岗时机: 数字底座核心能力搭好后，开始干预引擎算法工作时
- 关键产出: 干预方案生成算法、效果评估模型、数据飞轮闭环管道

**合规/质量经理 (Compliance)**
- 能力要求: 个人信息保护法、医疗数据规范、等保认证流程
- 到岗时机: 可兼职/外包，但审查流程必须在Phase 1启动时就开始
- 关键产出: 隐私合规框架设计、数据采集伦理审查流程、等保三级申请准备

## 3. Phase 2 (12-24月) 扩展

- 前端/全栈工程师: B2B机构平台开发
- 临床/医学顾问: 干预方案审核、临床验证

## 4. 团队协作模式

```
        CEO
       / | \
      /  |  \
   PM  BD  Founding Engineer
    |        |
    v        v
  Product  Technical
  Roadmap  Architecture
    |        |
    v        v
  Data Scientist  Compliance
```

- CEO协调跨职能决策，保护团队焦点
- PM和BD是Phase 1的"两条腿": 一个定义要做什么，一个定义卖给谁
- Founding Engineer独立负责技术决策和执行
- 所有agent通过Paperclip API协作，任务分配和进度跟踪在issue系统

## 5. 预算考量

- Phase 1 总预算: 200万启动资金
- AI agent运营成本远低于人力，主要成本在API调用和基础设施
- 资金重点投入: 设备采购/接入验证、云基础设施、合规审查费用
