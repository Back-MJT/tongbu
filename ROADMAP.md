## 项目概述

**主体**：宁津昕动智能科技有限公司
**英文名**：Ningjin Xindong Intelligent Technology Co., Ltd.
**核心方向**：
- 健康装备数字底座
- 个体化干预引擎
**愿景**：构建从数据采集→数字孪生→智能干预→效果评估的全链路健康科技平台
**竞争策略**：聚焦数字底座+干预引擎的平台层，以 B2B2C 切入，通过数据飞轮建立壁垒

## 路线图背景

### 市场痛点

- **市场碎片化**：健康设备品牌众多、协议不统一，数据孤岛严重
- **标准化缺失**：缺乏统一的健康数据采集、存储、应用标准
- **数据质量参差**：设备精度、数据连续性、完整性缺乏保障
- **干预效果难以量化**：现有健康管理方案缺乏个体化与效果闭环

### 核心价值定位

1. **数字底座标准化**：解决设备接入碎片化，提供统一数据底座
2. **个体化干预精准化**：基于多模态健康数据，实现"一人一策"干预方案
3. **"装备-数据-算法-干预"闭环**：从设备数据采集到干预效果评估的全链路能力

### 我们的解法

1. **数字底座**：提供统一设备接入层、数据管道与存储标准，消除碎片化
2. **干预引擎**：基于个体健康画像，生成动态化、精准化干预方案
3. **全链路闭环**：装备→数据→算法→干预→评估→优化的完整飞轮

### 竞争力体系

- **全栈技术能力**：设备协议接入 → 数据处理 → 算法建模 → 干预方案的全链路能力
- **三位一体闭环壁垒**：装备 + 数据 + 算法融合，形成难以复制的产品护城河
- **地方产业协同**：宁津本地政策资源与产业基础支撑，降低早期运营与获客成本

### 竞争策略

- **聚焦差异化**：不做单一硬件，专注数字底座+干预引擎的平台层
- **借力地方资源**：利用宁津本地产业基础和政策支持，降低早期获客成本
- **数据飞轮**：数据积累 → 模型优化 → 干预效果提升 → 更多数据 → 更强模型
- **标准先行**：推动行业数据标准与接口规范建立

### 市场切入

- **第一阶段（0-12 月）**：以 B2B 为主，优先对接健康管理机构、体检中心、康复医院，依托本地资源建立首批合作
- **第二阶段（12-24 月）**：B2B 规模化，拓展至 10+ 家机构建立标杆，为 B2B2C 打基础
- **第三阶段（24-36 月）**：B2B2C 模式跑通，通过机构网络触达终端消费者（健康管理需求人群），探索 2G 场景（区域健康管理、医保控费）
- **第四阶段（36-60 月）**：开放 PaaS 平台成为行业基础设施；2G 区域健康管理平台上线，成为政府/医保合作方

### 核心里程碑

| 阶段 | 时间线 | 核心目标 |
|------|--------|----------|
| 数字底座 MVP + 引擎验证（Phase 1） | 0-12 月 | 底座核心能力就绪，5+ 设备接入，干预引擎 v0.1 在 1-2 个场景闭环验证，1000+ 用户数据资产 |
| 干预引擎 v1.0 + B2B 规模化（Phase 2） | 12-24 月 | 干预引擎覆盖 3+ 场景，10+ 家 B2B 机构标杆建立，数据飞轮稳定运转 |
| B2B2C + 标准先行（Phase 3） | 24-36 月 | 10 万+ 活跃用户，30+ B2B 机构，正毛利，2G 场景探索启动 |
| 生态与平台化 + 2G 平台（Phase 4） | 36-60 月 | 行业 PaaS 平台，数字孪生人体建模，2G 区域健康管理平台上线，行业标准制定者 |

## Phase 1：数字底座基础设施 + 干预引擎 v0.1（启动期，0-12 个月）

> **战略重心**：先搭底座，再建引擎。数字底座必须在干预引擎之前完成核心能力建设。
> **市场策略**：依托宁津本地产业资源，优先对接 2-3 家健康机构/康复医院，以 B2B 形式积累早期验证数据。

### 1.1 设备接入层

- [ ] 多协议设备接入网关（BLE/Wi-Fi/Zigbee 适配）
- [ ] 设备抽象模型定义（统一数据 schema）
- [ ] 实时数据管道（消息队列 + 时序数据库）
- [ ] 设备发现/配对/OTA 升级流程
- [ ] 设备健康状态监控与故障预警

### 1.2 数据基础设施

- [ ] 个人健康数据模型标准化（遵循 HL7 FHIR 标准）
- [ ] 数据存储方案选型（时序DB + 关系型 + 对象存储）
- [ ] 隐私合规框架设计（个人信息保护法 / 医疗数据规范）
- [ ] 数据质量监控与异常检测（缺失率/漂移率/延迟告警）
- [ ] 数据脱敏与访问控制机制

### 1.3 个体化干预引擎 v0.1

- [ ] 个体健康画像构建（多维特征工程：生理指标、行为数据、环境因素）
- [ ] 基线评估模型（个体健康状况量化评分）
- [ ] 干预方案生成引擎 v0.1（规则引擎为主，ML 为辅）
- [ ] 效果评估与 A/B 测试框架
- [ ] 反馈学习闭环（干预效果 → 模型更新）

### 1.4 首批验证场景

> 选择 1-2 个垂直场景，确保干预效果可量化，为后续场景扩展建立方法论。

- [ ] **运动干预**：个性化运动处方生成（强度/频率/类型/时长）
- [ ] **睡眠干预**：睡眠节律优化方案

### 1.5 MVP 验证

- [ ] 选定 3-5 款核心健康设备完成接入验证
- [ ] 端到端数据链路：设备→云端→仪表盘
- [ ] 完成首批种子数据采集（≥1000 人/30 天）
- [ ] 验证至少 1 个干预场景的效果闭环（干预前后指标可比对）
- [ ] 数据质量标准建立（设备精度、采集频率、断点续传保障）

### 1.6 数据飞轮启动

> 飞轮机制是核心竞争壁垒：更多数据 → 更好模型 → 更强干预 → 更多用户 → 更多数据

- [ ] 建立数据采集→模型更新→效果评估的自动化闭环管道
- [ ] 定义数据资产积累指标（用户数、设备连接数、数据维度数、干预效果样本量）
- [ ] 设计数据标注协议（干预效果标签化，为后续 ML 训练提供标注数据）
- [ ] 初步验证数据飞轮效果（积累一定量数据后，模型精度是否有提升趋势）

**里程碑**：数字底座核心能力就绪 + 干预引擎 v0.1 在 1-2 个场景验证通过 + 1000+ 用户数据资产沉淀 + 数据飞轮首环跑通

## Phase 2：干预引擎 v1.0 + B2B 规模化（12-24 个月）

> **战略重心**：从"能跑通"到"能规模化"。引擎从规则为主转为 ML 为主，干预场景扩展到 3+ 个（心血管/糖尿病/康复），产品形态升级为可部署的 B2B 平台。
> **市场策略**：B2B 渠道拓展至 10+ 家机构建立标杆，为后续 B2B2C 铺路。借力宁津本地产业资源降低早期获客成本，推动数据飞轮运转。优先对接健康管理机构、体检中心、康复医院。

### 2.1 干预算法升级

- [ ] 个体健康画像 v2.0（引入时序建模、多模态融合）
- [ ] 干预方案生成引擎 v1.0（ML 为主，规则引擎为辅）
- [ ] 动态干预调整（根据实时数据调整方案参数）
- [ ] 效果评估体系标准化（可量化、可复现的干预效果评估模板）
- [ ] 反馈学习闭环 v2.0（自动化模型更新流程）

### 2.2 新增干预场景

> 基于 Phase 1 验证的方法论，扩展到更多场景。优先选择数据充足、验证路径清晰的场景。

- [ ] 营养干预：基于体况的饮食建议
- [ ] 压力管理：心理状态监测与干预提醒
- [ ] 慢病管理场景（心血管/糖尿病等，根据临床合作伙伴需求确定）

### 2.3 B2B 机构平台

- [ ] 机构管理后台（机构级健康数据看板）
- [ ] 健康师工作台（方案审核、个案管理、效果追踪）
- [ ] 批量干预方案配置能力
- [ ] 机构客户自助分析报告生成
- [ ] 干预方案专家审核工具

### 2.4 平台能力增强

- [ ] 干预方案版本管理与可追溯性
- [ ] 方案效果可视化仪表盘
- [ ] 多机构数据隔离与租户管理
- [ ] 运营数据看板（机构/设备/用户维度监控）
- [ ] API 网关与第三方服务集成框架

**里程碑**：干预引擎 v1.0 上线，支持 3+ 场景 + 10 家 B2B 机构客户 + 标杆案例建立

## Phase 3：B2B2C 规模化 + 标准先行（24-36 个月）

> **战略重心**：从 B2B 到 B2B2C，通过机构网络触达终端消费者，建立用户规模和收入。同时启动 2G（政府/医保）场景探索，推动行业数据标准与接口规范，掌握标准制定话语权。
> **市场策略**：依托 Phase 2 建立的 B2B 机构网络，通过 B2B2C 模式获取 C 端用户（健康管理需求人群）。开放 API 吸引更多设备和服务接入。同步探索区域健康管理平台、医保控费场景。利用已积累的数据优势参与行业标准制定。

### 3.1 消费端产品

- [ ] C 端 APP / 小程序
  - 个人健康看板（数据可视化、趋势分析）
  - 每日干预任务推送与提醒
  - 进度追踪与成就系统
  - 健康资讯与知识服务
- [ ] 用户增长体系（分享、推荐、社交激励）

### 3.2 规模化架构

- [ ] 多租户架构全面完善
- [ ] 高并发数据管道优化（万级设备同时在线）
- [ ] 边缘计算节点部署（区域化部署方案）
- [ ] 数据分片与水平扩展能力
- [ ] SLA 保障与灾备机制

### 3.3 开放平台

- [ ] 第三方设备接入 API（SDK + 开发者文档）
- [ ] 第三方服务市场（营养咨询、运动指导等增值服务接入）
- [ ] 数据合作与研究机构数据服务（脱敏数据服务）

### 3.4 合规与认证

- [ ] 医疗器械相关资质申请（如适用）
- [ ] 信息安全等级保护认证（三级等保）
- [ ] 数据出境合规评估（如涉及跨境业务）
- [ ] 算法伦理审查与透明化机制

### 3.5 标准制定与行业影响

> 标准先行是竞争策略的核心。利用 Phase 2 积累的数据规模和标杆案例，推动行业标准落地。

- [ ] 发布健康装备数据接入规范白皮书
- [ ] 参与或主导 1-2 项行业/团体标准起草
- [ ] 开源设备适配层 SDK，降低第三方设备接入成本
- [ ] 建立"昕动标准"认证体系（兼容昕动数字底座的设备认证标识）

### 3.6 商业化指标

- [ ] 月活用户（MAU）≥ 100,000
- [ ] 用户留存率（30 日）≥ 40%
- [ ] B2B 客户数 ≥ 30 家
- [ ] 实现正毛利（里程碑确认）

**里程碑**：产品正式商用，B2B2C 跑通，10 万+ 活跃用户，正毛利

## Phase 4：生态与平台化 v3.0+（长期愿景，36-60 个月）

> **战略重心**：从产品到平台，从平台到生态，成为健康装备数字底座行业标准制定者。
> **市场策略**：通过 PaaS 平台聚合行业需求，利用数字孪生技术打开新增长曲线。以开放策略巩固数据飞轮壁垒。

### 4.1 行业 PaaS 平台

- [ ] 健康装备数字底座开放为行业 PaaS 平台（标准输出+设备接入 SaaS）
- [ ] 开发者门户与开放 API 市场（第三方设备/服务商自助接入）
- [ ] 多租户 PaaS 管理控制台（计费/监控/权限/配额）
- [ ] 行业解决方案模板库（可复用的场景化干预方案包）

### 4.2 数字孪生与前沿技术

- [ ] 数字孪生人体建模（个体全生命周期健康模拟与预测）
- [ ] AI 大模型驱动的智能健康助手（对话式健康指导）
- [ ] 个体化干预引擎对接更多垂直场景（康复/心理健康/职业病/母婴等）

### 4.3 生态建设

- [ ] 产业联盟与合作伙伴生态建设
- [ ] 推动行业数据标准与接口规范制定（成为国标/行标参与者）
- [ ] 学术合作与科研产出（高水平论文+专利构建技术护城河）

### 4.4 2G 区域健康管理平台

> 基于 Phase 3 探索成果，正式上线面向政府和医保的解决方案。
> 目标市场：区域健康数据平台、医保控费场景、公共卫生管理。

- [ ] 区域级健康管理平台解决方案（多机构/多设备统一接入）
- [ ] 医保控费数据分析模块（干预效果与医疗费用关联分析）
- [ ] 公共卫生监测仪表盘（区域健康趋势、流行病预警）
- [ ] 政府/医保客户专属合规与安全认证
- [ ] 建立 2G 标杆案例（1-2 个地级市合作项目）

**里程碑**：行业 PaaS 平台上线 + 数字孪生能力就绪 + 行业标准制定者
